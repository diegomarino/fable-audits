# fable-audits - adversarial audit specs - fetch & run
#
# Six audit specs plus two companions, each fully self-contained in commands/:
# - audit-codebase.md: codebase/implementation audit.
# - audit-docs.md: documentation audit.
# - audit-process.md: end-to-end workflow/process audit.
# - audit-security.md: system-level security + supply-chain audit.
# - audit-ux.md: end-user experience audit of the shipped surfaces.
# - audit-change.md: diff-scoped change audit (working tree or --base <ref>).
# - audit-full.md: orchestrator -- one fresh subagent per audit, then one synthesized fixes backlog.
# - fix.md: fixing agent -- consumes a report/backlog by finding ID; human-gated, never touches git.
#
# Each file is frontmatter (for the Claude Code plugin) + the complete /goal-style
# spec body. The launcher below only NAMES the file; the agent opens it with its
# own Read tool, so spec size never hits the /goal limit.
#
# -- Install option A: Claude Code plugin --
#   /plugin marketplace add diegomarino/fable-audits
#   /plugin install fable-audits@fable-audits
#   Then: /fable-audits:audit-codebase, :audit-docs, :audit-process,
#         :audit-security, :audit-ux, :audit-change --base origin/main,
#         :audit-full, :fix docs/audits/<report>.md
#
# -- Run option B: zero-footprint single /goal (any harness with gh + shell) --
# One paste does everything: the agent clones the specs to /tmp (OUTSIDE the
# target repo -- copying them into the project would dirty the tree and trip
# the suite's clean-tree preflight) and executes the named spec. gh handles
# auth, so this works for private installs of this repo. Re-running re-clones
# fresh, so you always audit with the latest specs. Swap <SPEC> for one of:
#   audit-codebase  audit-docs  audit-process  audit-security
#   audit-ux        audit-change    audit-full  fix

/goal First run: rm -rf /tmp/fable-audits && gh repo clone diegomarino/fable-audits /tmp/fable-audits -- --depth 1. Then read /tmp/fable-audits/commands/<SPEC>.md in full and execute its instructions to the letter - that file is your task spec, not reference to summarize or improve. Ignore its YAML frontmatter; the body is the spec. Stay strictly within the scope it defines: make no change it doesn't authorize, and never commit, push, or touch git state in the project under audit (the /tmp clone above is the only git operation allowed). Not done until its Output/stop criteria are met exactly as written.

#
# For audit-change.md, name the diff target the same way, e.g.:
#   ...as written. Target: --base origin/main   (or: working tree)
